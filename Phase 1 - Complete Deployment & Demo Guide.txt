## ðŸš€ Part 1: Launch EC2 Instance (5 minutes)

### Step 1: Launch Instance

**Via AWS Console:**

1. **Go to EC2 Dashboard**
   - Navigate to: https://console.aws.amazon.com/ec2/

2. **Launch Instance**
   - Click "Launch Instance"
   - Name: `phase1-demo-server`

3. **Choose AMI**
   - **Option A:** Amazon Linux 2023 (recommended)
   - **Option B:** Ubuntu Server 22.04 LTS

4. **Instance Type**
   - Select: `t3.micro` (Free Tier eligible)

5. **Key Pair**
   - Create new or select existing
   - Download `.pem` file
   - Save to: `~/.ssh/phase1-demo.pem`

6. **Network Settings**
   - Create security group:
     - Name: `phase1-demo-sg`
     - Allow SSH (port 22) from **Your IP only**
     - Click "Add rule" â†’ Allow HTTP (port 8080) from **Your IP** (for future web UI)

7. **Configure Storage**
   - 8 GB gp3 (default is fine)

8. **Launch Instance**
   - Wait 2-3 minutes for instance to start
   - Note the **Public IPv4 address**

### Step 2: Prepare SSH Key

```bash
# On your local machine
chmod 400 ~/.ssh/phase1-demo.pem

# Test connection (replace with your IP)
ssh -i ~/.ssh/phase1-demo.pem ec2-user@YOUR_EC2_IP

# For Ubuntu, use:
ssh -i ~/.ssh/phase1-demo.pem ubuntu@YOUR_EC2_IP
```

## ðŸ“ž Troubleshooting

### Issue: "Permission denied"
```bash
# Make scripts executable
sudo chmod +x /opt/admin_dashboard/scripts/user.sh
sudo chmod +x /opt/admin_dashboard/scripts/lib/*.sh
```

### Issue: "Config file not found"
```bash
# Check config exists
ls -l /opt/admin_dashboard/config/user_mgmt.conf

# If missing, copy it
sudo cp config/user_mgmt.conf /opt/admin_dashboard/config/
```

### Issue: "jq: command not found"
```bash
# Amazon Linux
sudo yum install -y jq

# Ubuntu
sudo apt-get update && sudo apt-get install -y jq
```

### Issue: Unicode characters don't display
```bash
# Edit config
sudo nano /opt/admin_dashboard/config/user_mgmt.conf

# Change to:
USE_UNICODE="no"
```

### Issue: Tests fail
```bash
# Check logs
sudo cat /tmp/phase1_test_*.log
sudo tail -100 /var/log/user_mgmt.log

# Run individual test
sudo ./test_phase1.sh add-user
```
